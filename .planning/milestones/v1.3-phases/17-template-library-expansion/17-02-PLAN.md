Create `apps/_templates/crud-app/` directory and its Flask backend.

**app.py**: Flask app with SQLite CRUD operations. Structure:

1. **Header docstring** explaining the template (same style as flask-basic/app.py)
2. **Imports**: `os, sqlite3, flask (Flask, render_template, request, redirect, url_for, flash, g, jsonify)`
3. **App setup**: `PORT`, `APP_NAME` from env vars, `app.secret_key` set to a placeholder string
4. **Database config**: `DATABASE` path set to `os.path.join(os.path.dirname(__file__), "data.db")`
5. **Model definition section** — clearly commented with `# ── Model Definition ──` header:
   - A dict or comment block defining the sample model "Item" with fields: `id` (INTEGER PRIMARY KEY AUTOINCREMENT), `name` (TEXT NOT NULL), `description` (TEXT), `status` (TEXT DEFAULT 'active'), `created_at` (TIMESTAMP DEFAULT CURRENT_TIMESTAMP)
   - `CREATE_TABLE_SQL` as a module-level string constant for the Item table
   - Comment: "To adapt: change the table name, columns, and form fields. The CRUD routes follow the same pattern for any model."
6. **Database helpers**:
   - `get_db()` — returns `g._database` connection with `sqlite3.Row` row_factory, creates table if not exists
   - `close_db(exception)` — registered via `@app.teardown_appcontext`
   - `init_db()` — called on first request to ensure table exists
7. **CRUD routes**:
   - `GET /` → redirect to `/items`
   - `GET /items` → list all items, render `list.html`
   - `GET /items/<int:id>` → show single item, render `detail.html`, 404 if not found
   - `GET /items/new` → render empty `form.html` for create
   - `POST /items` → insert new item from form data, redirect to detail page
   - `GET /items/<int:id>/edit` → render `form.html` pre-filled with item data
   - `POST /items/<int:id>/edit` → update item, redirect to detail page
   - `POST /items/<int:id>/delete` → delete item, redirect to list
8. **API endpoints** (for JS-based interactions):
   - `GET /api/items` → return all items as JSON
   - `DELETE /api/items/<int:id>` → delete item, return JSON `{"ok": true}`
9. **Entry point**: `app.run(host="0.0.0.0", port=PORT, debug=False)`

Use parameterized queries for all SQL (no f-strings with user input). Flash messages for create/update/delete confirmations.

**requirements.txt**: Same pattern as other templates — comment about flask pre-installed.

Create the HTML templates, CSS, and JS for the CRUD app.

**templates/base.html**: Copy the exact pattern from `flask-basic/templates/base.html`:
- `{% if app_name %}<base href="/{{ app_name }}/">{% endif %}` in head
- Same topbar structure with `{% block heading %}` and `{% block nav %}`
- Nav block with links: `<a href="items">All Items</a>` and `<a href="items/new">New Item</a>`
- Flash messages block: `{% with messages = get_flashed_messages() %} ... {% endwith %}` rendered as dismissible notification bars above main content
- `{% block content %}{% endblock %}` in main
- `<script src="static/app.js"></script>` before closing body
- Dark theme stylesheet link: `<link rel="stylesheet" href="static/style.css">`

**templates/list.html**: Extends base.html.
- Block title/heading: "Items" (or dynamic from model name)
- If items list is empty: show a "No items yet" message with a "Create first item" button
- If items exist: render a `.data-table` with columns: Name, Status, Created, Actions
- Each row: item name links to `items/{{ item.id }}`, status shown as a colored badge, created_at formatted, actions column has Edit and Delete buttons
- Delete uses a small inline form with `method="POST"` to `items/{{ item.id }}/delete` (not JS) — include a `data-confirm` attribute for JS confirmation
- Item count shown above the table

**templates/form.html**: Extends base.html. Shared for create and edit.
- Block title: "New Item" or "Edit Item" based on whether `item` variable exists
- `<form method="POST" action="...">` — action is `items` for create, `items/{{ item.id }}/edit` for edit
- Fields: name (text input, required), description (textarea), status (select dropdown: active, inactive, archived)
- If editing, pre-fill values from `item` object
- Submit button: "Create" or "Save Changes"
- Cancel link back to list (or detail if editing)

**templates/detail.html**: Extends base.html.
- Block title: item name
- Card layout showing all fields: Name (h2), Description (paragraph), Status (badge), Created (formatted timestamp)
- Action buttons: Edit (link to `items/{{ item.id }}/edit`), Delete (form POST to `items/{{ item.id }}/delete`), Back to List
- Delete button should have a JS confirmation prompt

**static/style.css**: Dark theme matching existing templates. Use same CSS custom properties (--bg, --surface, --border, --text, --accent, --green, --orange, --red, --radius, --font). Include:
- Same reset, base, layout, topbar styles as flask-basic
- `.data-table` — full-width table with dark theme (striped rows, hover highlight, border-collapse)
- `.badge` — small colored pill for status (green for active, orange for inactive, muted for archived)
- Form styles: `.form-group` (label + input stacked), `.form-control` (dark-themed input/textarea/select), `.form-actions` (button row)
- Flash message styles: `.flash` bar with success (green border-left) and error (red border-left) variants
- `.empty-state` — centered message for empty lists
- Same `.btn` styles as other templates
- Responsive: table scrolls horizontally on mobile, form goes full-width

**static/app.js**: Minimal JS:
- Delete confirmation: `document.querySelectorAll('[data-confirm]')` → addEventListener on parent form submit → `confirm("Are you sure?")` → prevent default if cancelled
- Flash message auto-dismiss: fade out after 5 seconds
- Optional: fetch helper for API endpoints (same pattern as flask-basic)
