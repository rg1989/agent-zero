  <action>
Create `apps/_templates/dashboard-realtime/` — a Flask dashboard with Server-Sent Events (SSE) for live data streaming AND a fallback `/api/data` polling endpoint. This is distinct from the existing `flask-dashboard` (which only does polling): this template uses SSE as its primary data channel and features multiple chart types.

**app.py**: Flask app with:
- Standard PORT/APP_NAME from env vars
- `@app.route("/")` renders `templates/index.html` with `app_name=APP_NAME, title="Dashboard"`
- `@app.route("/api/stream")` — SSE endpoint using Flask `Response` with `mimetype="text/event-stream"` and a generator that yields `data: {json}\n\n` every 2 seconds. Data shape: `{"metrics": [...], "charts": {"line": {...}, "bar": {...}, "doughnut": {...}}}`. Use `random` module for sample data. Include `Cache-Control: no-cache` and `Connection: keep-alive` headers.
- `@app.route("/api/data")` — polling fallback returning the same data shape as a single JSON response (for clients that cannot use SSE)
- Sample data generator function shared between SSE and polling endpoints producing: 4 metric cards (Total, Active, Rate%, Errors), a line chart dataset (12 time points), a bar chart dataset (6 categories), and a doughnut chart dataset (4 segments)
- Listen on `host="0.0.0.0"`, `port=PORT`, `debug=False`

**templates/index.html**: Self-contained dashboard page with:
- `{% if app_name %}<base href="/{{ app_name }}/">{% endif %}` in head
- Chart.js v4 from CDN: `https://cdn.jsdelivr.net/npm/chart.js@4/dist/chart.umd.min.js`
- Topbar with title, status dot (connected/disconnected/error), and last-updated timestamp
- Metrics grid (auto-fit responsive grid, same `.metric-card` pattern as flask-dashboard)
- THREE chart sections in a responsive grid:
  1. Line chart (time series — full width)
  2. Bar chart and Doughnut chart (side by side, 2-column grid)
- Inline `<script>` that:
  - Opens `EventSource("api/stream")` for live updates
  - On `message` event: parse JSON, call renderMetrics(), renderLineChart(), renderBarChart(), renderDoughnutChart()
  - On `error`: update status dot to error, fall back to polling `/api/data` every 5 seconds
  - Chart instances stored in variables, updated via `.data = newData; .update("none")` (not destroyed/recreated)
  - Status dot: green + glow when SSE connected, orange pulse when reconnecting, red on error

**static/style.css**: Dashboard-optimized dark theme. Use the same CSS custom properties as flask-dashboard (--bg, --surface, --border, --accent, etc). Include:
- Same topbar, layout, status-dot, metric-card, card styles as flask-dashboard
- `.charts-grid` — 2-column responsive grid for bar + doughnut charts
- `.chart-full` — full-width line chart card
- Chart container heights sized appropriately (line: 280px, bar/doughnut: 240px)
- Mobile responsive: charts stack vertically on small screens

**requirements.txt**: Same pattern — flask pre-installed comment, empty extras list.

IMPORTANT: The SSE generator must include `yield` with proper SSE format: `f"data: {json.dumps(payload)}\n\n"`. Use `time.sleep(2)` between yields in the generator. The Response must have `mimetype="text/event-stream"`.
