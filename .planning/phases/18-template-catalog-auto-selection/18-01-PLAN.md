Create `apps/_templates/_CATALOG.md` — a structured catalog listing all 7 templates. Use YAML-in-markdown format so it's both human-readable and parseable by the agent.

For each template, include these fields:
- `name`: directory name (e.g., `flask-basic`)
- `description`: one-line description
- `use_cases`: list of 3-5 typical use cases / user requests this template fits
- `pick_when`: concise "pick this when..." criteria for auto-selection
- `start_command`: the command to start the app (e.g., `python app.py`)
- `has_backend`: true/false (Python logic vs static serving)
- `has_database`: true/false
- `key_features`: list of notable features

The 7 templates and their details:

1. **flask-basic** — General web apps with Python backend logic. Pick when: forms, pages, dynamic content, custom API endpoints, anything not covered by specialized templates. Start: `python app.py`. Backend: yes. DB: no. Features: Flask routes, Jinja2 templates, base layout, dark theme, fetch helper.

2. **flask-dashboard** — Metrics dashboards with polling data refresh. Pick when: monitoring, metrics, single chart type, polling refresh is sufficient. Start: `python app.py`. Backend: yes. DB: no. Features: Chart.js line chart, metric cards, /api/data endpoint, polling refresh, psutil-ready.

3. **static-html** — Pure front-end with no Python backend logic. Pick when: D3/Plotly visualizations, static data displays, external API consumers, no server-side processing. Start: `python serve.py`. Backend: no. DB: no. Features: serve.py static server, Chart.js/D3/Plotly options, base tag injection.

4. **utility-spa** — Lightweight single-page tool (calculator, converter, text processor). Pick when: input-process-output tools, calculators, text transformers, unit converters, viewers, no backend logic needed. Start: `python serve.py`. Backend: no. DB: no. Features: input/output layout, action buttons, copy-to-clipboard, vanilla JS.

5. **dashboard-realtime** — Real-time dashboard with SSE streaming and multiple chart types. Pick when: live data feeds, real-time monitoring, multiple chart types needed, SSE streaming preferred over polling. Start: `python app.py`. Backend: yes. DB: no. Features: SSE + polling fallback, Chart.js line/bar/doughnut, connection status indicator, metric cards.

6. **crud-app** — Data management app with SQLite database. Pick when: managing records/items/entries, list/detail/create/edit/delete operations, needs a database. Start: `python app.py`. Backend: yes. DB: yes (SQLite). Features: SQLite with get_db pattern, parameterized SQL, data table, shared form, flash messages, 404 handling.

7. **file-tool** — File upload, management, and conversion tool. Pick when: file uploads, file management, format conversions, media processing, document tools. Start: `python app.py`. Backend: yes. DB: no. Features: drag-and-drop upload, file-card grid, download endpoints, format conversion pipeline, size limits.
