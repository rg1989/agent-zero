Create `apps/_templates/file-tool/` directory and its Flask backend.

**app.py**: Flask app with file upload, listing, download, and conversion. Structure:

1. **Header docstring** explaining the template (same style as flask-basic/app.py): "file-tool — Flask app for file upload, management, and conversion."
2. **Imports**: `os, json, time, mimetypes, shutil` from stdlib; `flask (Flask, render_template, request, jsonify, send_from_directory, redirect, url_for, abort)`
3. **App setup**: `PORT`, `APP_NAME` from env vars. `UPLOAD_DIR` set to `os.path.join(os.path.dirname(__file__), "uploads")`. Create `UPLOAD_DIR` if it does not exist (`os.makedirs(UPLOAD_DIR, exist_ok=True)`). Set `app.config["MAX_CONTENT_LENGTH"] = 50 * 1024 * 1024` (50MB limit).
4. **Allowed extensions** — a set: `{"txt", "csv", "json", "md", "html", "css", "js", "py", "png", "jpg", "jpeg", "gif", "svg", "pdf", "zip"}`. Helper function `allowed_file(filename)` checks extension.
5. **Helper function** `get_file_info(filename)` — returns dict with: `name`, `size` (bytes), `size_human` (formatted KB/MB), `ext`, `mime_type` (from mimetypes.guess_type), `uploaded_at` (file mtime as ISO string).
6. **Page routes**:
   - `GET /` → render `index.html` with `app_name=APP_NAME, title="File Manager"`
7. **API routes**:
   - `POST /api/upload` — accepts `multipart/form-data` with field name `file` (single) or `files` (multiple). For each file: validate extension, save with `secure_filename` (from `werkzeug.utils`), return JSON with list of uploaded file info dicts. Return 400 if no file or invalid extension.
   - `GET /api/files` — list all files in UPLOAD_DIR, return JSON array of file info dicts sorted by upload time descending
   - `GET /api/download/<filename>` — serve file from UPLOAD_DIR using `send_from_directory` with `as_attachment=True`
   - `DELETE /api/files/<filename>` — delete file from UPLOAD_DIR, return `{"ok": true}`. Return 404 if not found.
   - `POST /api/convert/<filename>` — format conversion endpoint. Accept JSON body `{"format": "target_format"}`. Supported conversions:
     - `txt` → `json` (wrap text lines as JSON array)
     - `csv` → `json` (parse CSV to list of dicts using stdlib `csv` module)
     - `json` → `csv` (flatten JSON array of dicts to CSV using `csv.DictWriter`)
     - `txt` → `uppercase` (convert text content to uppercase, save as new file `{name}_upper.txt`)
     - Return the converted file info. If conversion not supported, return 400 with `{"error": "Unsupported conversion"}`.
8. **Entry point**: `app.run(host="0.0.0.0", port=PORT, debug=False)`

Use `werkzeug.utils.secure_filename` for all uploaded filenames. Add `import csv` and `import io` for CSV conversion.

**requirements.txt**: Same pattern — flask pre-installed comment. No extra deps needed (csv, json, mimetypes are all stdlib).

Create the HTML templates, CSS, and JS for the file tool.

**templates/base.html**: Copy the exact pattern from `flask-basic/templates/base.html`:
- `{% if app_name %}<base href="/{{ app_name }}/">{% endif %}` in head
- Same topbar structure
- `{% block content %}{% endblock %}` in main
- `<script src="static/app.js"></script>` and `{% block scripts %}{% endblock %}`

**templates/index.html**: Extends base.html. Single-page file manager layout:
- Block title/heading: "File Manager"
- **Drop zone** at top: a large dashed-border area (`.dropzone`) with centered text "Drop files here or click to browse" and a hidden `<input type="file" multiple>` triggered by clicking the zone. The zone should also accept a file picker button.
- **Upload progress area** (`.upload-progress`, hidden by default): shows filename and progress bar during upload
- **File listing** below: a `.file-list` container populated by JS. Each file rendered as a `.file-card` (card style) showing:
  - File icon (emoji based on extension: text files, images, code, archive)
  - File name (truncated if long)
  - File size (human readable)
  - Upload date
  - Action buttons: Download (link to `api/download/{name}`), Convert (if applicable — shown only for txt, csv, json files), Delete (with confirmation)
- **Empty state**: "No files uploaded yet" message when list is empty
- **Status toast area** (`.toast-container`): fixed bottom-right for success/error messages

**static/style.css**: Dark theme matching existing templates. Same CSS custom properties. Include:
- Same reset, base, layout, topbar as flask-basic
- `.dropzone` — large dashed-border area, centered content, hover/active states (border color change to accent), transition. When dragging over: `border-color: var(--accent); background: rgba(0,242,254,0.05)`
- `.dropzone.dragover` — active drag state class
- `.file-list` — grid of file cards (auto-fit, minmax 280px)
- `.file-card` — card style with file icon, name, metadata row, action buttons
- `.file-icon` — large emoji display
- `.progress-bar` — dark track with accent-colored fill, animated width
- `.toast-container` — fixed bottom-right
- `.toast` — notification card with slide-in animation, success (green) and error (red) variants
- Same `.btn` styles as other templates, plus `.btn-icon` (small icon-only button)
- `.empty-state` — centered message
- Responsive: file cards stack on mobile, dropzone takes full width

**static/app.js**: Full file management client-side logic:

1. **Drop zone setup**:
   - Get references to `.dropzone` and hidden file input
   - `dragover`/`dragenter`: prevent default, add `.dragover` class
   - `dragleave`/`drop`: remove `.dragover` class
   - `drop`: get `e.dataTransfer.files`, call `uploadFiles(files)`
   - Click on dropzone: trigger hidden file input click
   - File input `change`: call `uploadFiles(input.files)`

2. **Upload function** `uploadFiles(fileList)`:
   - Create `FormData`, append each file with key `files`
   - `fetch("api/upload", { method: "POST", body: formData })` — no Content-Type header (browser sets multipart boundary)
   - Show upload progress area during upload (hide on complete)
   - On success: show toast "N file(s) uploaded", refresh file list
   - On error: show toast with error message

3. **File listing** `loadFiles()`:
   - `fetch("api/files")` → parse JSON → render file cards into `.file-list`
   - Each card: icon (emoji by extension), name, size, date, action buttons
   - Wire download buttons as `<a href="api/download/{name}">`
   - Wire delete buttons to `deleteFile(name)` with confirm
   - Wire convert buttons to show a simple dropdown/prompt for target format, then call `convertFile(name, format)`
   - Call `loadFiles()` on DOMContentLoaded

4. **Delete function** `deleteFile(name)`:
   - `confirm("Delete {name}?")` → `fetch("api/files/{name}", { method: "DELETE" })`
   - On success: toast + reload list

5. **Convert function** `convertFile(name, format)`:
   - `fetch("api/convert/{name}", { method: "POST", headers: {"Content-Type": "application/json"}, body: JSON.stringify({format}) })`
   - On success: toast "Converted to {format}" + reload list
   - On error: toast with error

6. **Toast helper** `showToast(message, type="success")`:
   - Create toast element, add to `.toast-container`, auto-remove after 3 seconds
   - Types: "success" (green border), "error" (red border)

7. **File icon helper** `getFileIcon(ext)`:
   - Return emoji by extension category: txt/md/csv → document, py/js/html/css → code, png/jpg/gif/svg → image, zip → archive, pdf → document, default → generic file
